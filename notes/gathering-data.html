11.1 Gathering data

11.1.1 Deliberately available

There are a variety of different options in terms of gathering data. We may find government datasets (opendatatoronto), censuses (distinguished given their importance), private sector datesets (Airbnb). These are datasets that have been made available deliberately for these purposes.

It may be that a survey has been run in order to generate this data. If that’s the case then we need to be aware of:

  • What is the relevant population?
  • How was the sampling frame identified?
  • What method of sampling are they using? If probabilistic then is it:
    • Random sampling? (Everyone the same chance.)
    • Stratified random sampling? (Everyone has the same chance within certain groups.)
    • Cluster sampling? (Everyone has the same chance within a given cluster, and no chance outside that cluster.)
  • If non-probabilistic, then what types of errors can occur and is there anything that you can do about it?

When we are thinking about these aspects, we need to think about how they could affect our results and if there is any way of adjusting for it. Either way, these aspects always need to be mentioned.

11.1.2 By product

There are also datasets that can be created as a by-product of someone trying to achieve another aim. For instance, a retailer may have a website with their products and their prices. That has not been created deliberately as a source of data, but we can scrape it to create a dataset.

There are a few general principles that I try to follow when it comes to this:

  1. Try to use an API.
  2. Check robots.txt.
  3. Slow down (why not only have it run once a minute if it can just run in the background?).
  4. Considering the timing (if it’s a retailer then why not set your script to run overnight).
  5. Only scrape once (save the data as you go and monitor where you are up to).
  6. Don’t republish the data you scraped (cf datasets that create based off it).
  7. Take ownership (add contact details to your scripts, don’t hide behind VPNs, etc)
